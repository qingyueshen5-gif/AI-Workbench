# 上线硬骨头3A-R2.1：Managed Proxy 机制落地

日期：2026-07-23

## 做了什么

- 新增 `managed-proxy/` Cloudflare Workers 服务骨架，包含安装注册、token 刷新、模型白名单、D1 用量记录、限额检查和 OpenAI-compatible `/v1/chat/completions` 转发。
- `model-proxy.mjs` 新增 `managed_remote` 路径：安装版没有本机 `DEEPSEEK_API_KEY` 时可通过 `MANAGED_PROXY_URL` / `AIW_MANAGED_PROXY_URL` 连接远程 Managed Proxy。
- 本机只保存安装实例 token，不保存 DeepSeek shared key；Windows 下 token 使用 DPAPI 加密保存到运行时 `config/managed-proxy.json`。
- 新增 `verify:managed-proxy-production`，用本地 mock Managed Proxy 验证 18800 注册、转发、token 保存和日志脱敏。

## 为什么这样做

正式共享 key 不能进入安装包、用户电脑、本机 `.env`、进程参数或日志。R2.1 的最小正确边界是：客户端和员工继续打本机 18800，本机 18800 只拿实例 token，真实 DeepSeek key 只放远程 Worker Secret。
