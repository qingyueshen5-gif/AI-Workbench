# 上线硬骨头3A-R2.1：Cloudflare 生产部署

日期：2026-07-24

## 做了什么

- 使用 Wrangler 登录 Cloudflare，创建 D1 数据库 `aiw-managed-proxy`。
- 将 D1 `database_id` 写入 `managed-proxy/wrangler.jsonc`，执行 `managed-proxy/schema.sql`，确认 `installations`、`daily_usage`、`revoked_tokens` 三张表可读写。
- 通过 Cloudflare Secret 配置 `DEEPSEEK_API_KEY`、`TOKEN_SIGNING_SECRET`、`INSTALLATION_HASH_SALT`；Secret 值未打印、未写文件、未进 Git。
- 部署 Worker 到 `https://ai-workbench-managed-proxy.qingyueshen5.workers.dev`。
- 将公开 Worker URL 内置为安装版默认 Managed Proxy URL；本机 `DEEPSEEK_API_KEY` 仍优先，安装版禁止回退本机 shared key 作为生产方案。
- 重新构建 v0.4.6 候选安装包并真实安装，安装版在无本机 Key 环境下通过生产 Worker 完成模型对话。
