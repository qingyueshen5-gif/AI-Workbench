# 文档基准纠偏与防漂移机制

日期：2026-07-24

## 本轮目标

统一 AI Workbench 当前文档基准，建立事实单一归属规则、Handoff 自动生成机制和文档一致性校验，防止后续新对话继续读到旧版本或冲突路线。

## 本轮边界

- 只修改文档、协议和文档工具。
- 不修改产品功能代码、安装包、Release、Cloudflare 配置、Worker Secrets 或模型白名单。
- 不执行产品资产备份、电脑清理、删除文件、卸载软件或后续新功能开发。

## 发现的旧状态和冲突

- `CONTEXT.md` 仍把 v0.2.0 描述为当前版本，已纠正为 v0.4.6 Alpha，并将 v0.2.0 保留为历史记录。
- `AI-Workbench-Handoff.md` 人工复制了 `CURRENT_PROGRESS_AUDIT.md`、`PRODUCT.md`、`VISION.md`、`CURRENT_TASK.md` 和 research 文档的大段正文，存在再次漂移风险。
- `CURRENT_TASK.md` 中存在历史路线图和多个下一步表述，已改为只描述当前任务和最近完成事实，下一步统一引用 `NEXT_STEP.md`。
- `LAUNCH.md` 的上线最小集未明确区分已完成与未完成，已更新为当前状态。

## 事实归属规则

- 当前版本：唯一权威是 `package.json` 的 `version`。
- 当前唯一下一步：唯一权威是 `NEXT_STEP.md`。
- 已完成/未完成能力：唯一权威是 `CURRENT_PROGRESS_AUDIT.md`。
- 产品方向：`PRODUCT.md`、`VISION.md`、`PRINCIPLES.md`、`DECISIONS.md` 分工承载。
- Release 事实：以 GitHub Release 和 `verification/3b-release/summary.json` 为准。
- 历史记录：`TASKLOG.md`、`CHANGELOG.md`、`tasks/`、`verification/`、`research/` 保留当时状态，不为统一当前口径而篡改历史。
- Handoff：只承载自动生成的最小快照和权威文件索引，不手工复制源文档正文。

## Handoff 方案

新增 `scripts/generate-handoff.mjs`，从以下权威来源生成 `AI-Workbench-Handoff.md` 标记区：

- `package.json`
- `NEXT_STEP.md`
- `CURRENT_PROGRESS_AUDIT.md`
- `PRODUCT.md`
- `VISION.md`
- `PRINCIPLES.md`
- `DECISIONS.md`
- `verification/3b-release/summary.json`

Handoff 固定区保留新对话交接规则和权威文件索引；自动区只保留足够新对话独立看懂项目的短快照。

## 校验脚本检查项

新增 `scripts/verify-docs-consistency.mjs`，检查：

- `package.json.version` 与 `CONTEXT.md` 版本标记区一致。
- 当前状态文件没有把 v0.2.0 描述为当前版本。
- `NEXT_STEP.md` 是唯一下一步来源，其他当前状态文件没有冲突下一步。
- 模型分层、手机端、完整多 Agent 调度、情报流水线、跨网站复杂执行、国际化和区域合规未被标记为已完成。
- ③A、③B、v0.4.6 Release 状态与验收 summary 一致。
- Handoff 自动生成区与权威文件一致且幂等。
- Handoff 不含手工复制的大段源文档正文，并包含版本、架构、下一步、分工、交接方法。
- 必读文件全部存在。
- 新增脚本和当前状态文档不含真实 Key、Token、Cookie、Secret 或用户数据。
