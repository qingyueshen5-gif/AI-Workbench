真实根因：

- `actions-build.log` 显示 `electron-builder` 在 Actions 上失败：

```text
The specified electronDist does not exist: D:\a\AI-Workbench\AI-Workbench\node_modules\electron\dist
```

- `package.json` 里写死了：

```json
"electronDist": "node_modules/electron/dist"
```

- GitHub Actions 的 `npm ci` 后该目录不存在，electron-builder 因显式 `electronDist` 直接失败，没有产出 `release-v0.4.6-installer/AI-Workbench-Setup-v0.4.6-x64.exe` 或 `win-unpacked`。
- `preflight-summary.json` 因此记录：
  - `artifactExists: false`
  - `unpackedExists: false`
  - `artifactPath: ""`
  - `sha256: ""`
  - `fiveCriteria.e_readinessReportComplete: false`
- 同时 `scripts/verify-install-release.mjs` 在 artifact 缺失时仍会读取仓库中旧的 `nsis-install-uninstall.json`，造成报告里出现“安装 passed / 卸载 passed”但安装包本身 missing 的混乱证据。
