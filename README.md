# AI Workbench

一个本地运行的极简 AI 工作台 MVP，用来跑通「聊天输入 → AI提炼结构化数据 → 系统留痕 → 展示进度」的最小闭环。

当前版本是 `v0.2.0 聊天为中心`。这是 Phase 3 的第一步：把 v0.1.1 的多页面表单收缩成一个聊天入口，让 DeepSeek 从用户自然语言里提炼今日目标、任务和偏好。

## 当前功能

- 主界面：连续聊天流，用户直接说今天想做什么。
- 自动提炼：发送聊天后调用 DeepSeek，识别今日目标、任务和用户偏好。
- 右侧栏：查看今日目标、任务列表、任务详情、历史记录和失败原因搜索。
- 兜底修正：AI 判断错时，用户仍可手动修改任务状态、负责人、备注和失败原因。
- 系统错误日志：记录模型调用等系统级错误，并支持按关键词搜索。

## 关键规则

任务状态可以是 `待开始`、`进行中`、`已完成`、`失败`。任何任务被保存为 `失败` 时，必须填写失败原因；这是当前版本的错误记忆库雏形。

## 技术栈

- React + Vite
- Tailwind CSS
- Node.js 本地 API
- 本地 JSON 文件存储：`data/workbench.json`

## 启动方法

首次运行先安装依赖：

```bash
npm install --cache .npm-cache
```

启动开发环境：

```bash
npm run dev
```

然后打开：

```text
http://127.0.0.1:5173
```

`npm run dev` 会同时启动：

- 前端 Vite 服务：`http://127.0.0.1:5173`
- 本地 API 服务：`http://127.0.0.1:8787`

## 验证方法

1. 在聊天输入框发送“我今天想把登录页面做完，默认负责人是Codex”。
2. 确认右侧今日目标、任务列表或偏好被自动提炼更新；不确定内容会在聊天消息下方显示确认按钮。
3. 在右侧任务详情里切换负责人、备注和状态，确认刷新页面后数据仍保留。
4. 将任务保存为失败时，先不填失败原因，确认页面阻止保存；填写失败原因后再保存。
5. 在右侧历史和错误里按关键词搜索失败原因，确认能找到对应失败任务。

## DeepSeek 配置

在项目根目录创建 `.env`，写入：

```text
DEEPSEEK_API_KEY=你的DeepSeek密钥
```

`.env` 已被 `.gitignore` 忽略，不会提交到 GitHub。配置后启动项目，聊天发送会自动调用 DeepSeek 做信息提炼。
