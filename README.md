# AI Workbench

一个本地运行的极简 AI 工作台 MVP，用来跑通「发指令 → 系统留痕 → 展示进度」的最小闭环。

## 当前功能

- 首页：填写今天目标，新增和勾选今日任务，查看今日完成进度。
- 聊天页：发送消息，保留消息记录，把消息手动标记为任务。
- 任务状态页：查看任务列表，切换状态，选择负责人，记录备注。
- 历史记录页：按天查看目标和任务完成情况，搜索历史失败原因。

## 关键规则

任务状态可以是 `待开始`、`进行中`、`已完成`、`失败`。任何任务被保存为 `失败` 时，必须填写失败原因；这是当前版本的错误记忆库雏形。

## 技术栈

- React + Vite
- Tailwind CSS
- Node.js 本地 API
- 本地 JSON 文件存储：`data/workbench.json`

## 启动方法

首次运行先安装依赖：

```bash
npm install --cache .npm-cache
```

启动开发环境：

```bash
npm run dev
```

然后打开：

```text
http://127.0.0.1:5173
```

`npm run dev` 会同时启动：

- 前端 Vite 服务：`http://127.0.0.1:5173`
- 本地 API 服务：`http://127.0.0.1:8787`

## 验证方法

1. 打开首页，填写今天目标，新增一个今日任务并勾选完成，确认进度百分比变化。
2. 打开聊天页，发送一条消息，点击「标记为任务」，确认任务同步到任务状态页。
3. 打开任务状态页，切换负责人、备注和状态，确认刷新页面后数据仍保留。
4. 将任务保存为失败时，先不填失败原因，确认页面阻止保存；填写失败原因后再保存。
5. 打开历史记录页，按关键词搜索失败原因，确认能找到对应失败任务。
