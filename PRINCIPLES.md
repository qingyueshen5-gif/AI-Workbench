# PRINCIPLES.md — 设计原则

## 数据原则

1. **纯数据层，不做研判** — 采集和估值是数据工作，投资判断由人来做
2. **不编造数据** — 所有估值输入来自公开数据源，缺数据宁可留空
3. **多层容错** — 一个数据源挂了自动切换备用源
4. **可追溯** — 每个估值结果都能追溯到原始输入

## 工程原则

1. **版本迭代留存** — 每次重大判断/策略变更加版本号，不做覆盖（V1/V2/V3 各自留存）
2. **简单优先** — Python 脚本 > 重型框架，能跑就行
3. **幂等性** — 重复运行不会产生副作用
4. **定时自动化** — Cron 调度，到点就跑，不用人管

## 投资哲学（用户侧）

1. **三年时间框架** — 不做日内/短线，以年为单位思考
2. **周期思维** — 关注猪周期、行业周期、市场周期
3. **安全边际** — 估值留够冗余，不追高
4. **能力圈** — 深耕熟悉的行业和标的

## 信息源优先级

1. 官方公告（年报、季报、农业农村部数据）
2. 行业数据库（Finviz、Sina 财经、腾讯财经）
3. 市场信号（猪粮比、A-H 溢价率）
4. 社交媒体（13 博主观测、叙事追踪）
