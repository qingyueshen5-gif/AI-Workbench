# Hermes One 商业版产品形态对标

> 依据已提供的界面观察整理，用于 AI Workbench 后续产品路线对标。

## 1. 员工管理体系

Hermes One 的核心产品形态是“AI 员工”管理后台。用户可以创建不同员工，为每个员工配置职责、能力边界、模型、工具权限和上线状态。员工不是单次对话，而是可持续运行、可被分配任务、可接入外部通道的工作单元。

典型能力包括：

1. 创建员工：按岗位或任务类型创建代理，例如客服、运营、内容、销售、助理。
2. 配置员工：设置提示词、知识、工具、权限、运行规则和失败处理方式。
3. 选择模型：为不同员工绑定不同模型，按成本、速度、能力做差异化配置。
4. 上线员工：员工配置完成后进入可运行状态，可被任务、通道或用户消息触发。

我们的对标策略:功能内置化,用户只见一个页面

## 2. 多平台通道

Hermes One 展示出多通道接入形态，重点是把同一个 AI 员工发布到用户真实使用的平台里。微信、飞书、Telegram、QQ、抖音、小红书等平台通过扫码或授权方式接入，让代理可以在外部消息入口里工作。

典型能力包括：

1. 微信接入：通过扫码或账号授权，把员工连接到微信会话。
2. 飞书接入：面向办公协作场景，承接团队消息、审批和任务流。
3. Telegram 接入：面向海外或技术用户，支持 Bot 式交互。
4. QQ 接入：覆盖国内 IM 场景。
5. 抖音和小红书接入：面向内容、私信、评论、线索收集等运营场景。

我们的对标策略:功能内置化,用户只见一个页面

## 3. 技能/插件生态

Hermes One 的技能/插件生态接近应用商店。用户可以从技能库中选择能力模块，把它们挂载到员工身上，减少从零配置复杂工具链的成本。

典型能力包括：

1. 技能库浏览：按场景展示可用技能，例如搜索、文件、表格、网页、CRM、内容发布。
2. 一键启用：用户选择技能后，系统完成工具绑定、权限说明和运行配置。
3. 技能组合：一个员工可组合多个技能，形成岗位能力包。
4. 第三方扩展：生态成熟后可支持外部开发者提交技能。

我们的对标策略:功能内置化,用户只见一个页面

## 4. 任务编排

Hermes One 不只做聊天，还强调任务编排。任务可以被定时触发，也可以由消息、外部事件或流程节点触发，形成自动化工作流。

典型能力包括：

1. 定时任务：按时间周期运行，例如日报、巡检、内容更新、数据同步。
2. 自动化流程：把多个动作串成流程，例如收消息、查资料、写总结、发到群。
3. 状态追踪：展示任务是否执行、是否失败、失败原因和下一步。
4. 多员工协作：不同员工可以分工处理同一个流程里的不同节点。

我们的对标策略:功能内置化,用户只见一个页面

## 5. 记忆系统

Hermes One 的记忆系统承担代理长期上下文。它不仅保存代理自己的工作记忆，也保存用户画像，让员工在后续任务中理解偏好、历史、约束和关系。

典型能力包括：

1. 代理记忆：保存员工执行经验、失败原因、常用步骤和上下文。
2. 用户画像：保存用户偏好、身份、常用平台、工作习惯和重要约束。
3. 可复用上下文：新任务可以自动读取相关记忆，减少重复说明。
4. 记忆治理：需要支持查看、更新、删除和权限边界，避免错误记忆长期污染。

我们的对标策略:功能内置化,用户只见一个页面

## 产品结论

Hermes One 的商业版结构是“员工 + 通道 + 技能 + 编排 + 记忆”的完整生态。AI Workbench 的短期路线不应照搬复杂后台，而应把这些能力折叠到单一工作台体验里：用户只描述目标，系统内部完成员工选择、工具调用、记忆读取、通道同步和任务追踪。
